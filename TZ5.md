# 2. Границы MVP

## 2.1 Входит в MVP

- Регистрация, авторизация, роли и права
- Личный кабинет ученика, учителя, родителей
- **Библиотека системных курсов (один курс в MVP): «Нейросети для IT-проектов: начало»**
- Система курсов: системные и авторские курсы учителей
- Видеоуроки, тесты, прогресс
- ИИ-среда создания проектов:
  - Поле промта + выбор модели
  - Streaming-ответы ИИ посимвольно
  - Генерация Python-кода и сохранение версий
- Запуск проектов в Python-sandbox (внутри платформы)
- Библиотека шаблонов проектов и промтов
- Базовая аналитика (ученик / класс / админ)
- Уведомления (email + in-app)
- Родительский кабинет (общий на ребёнка, несколько взрослых)
- Ручное управление подписками и токенами администратором
- Интеграция с российскими LLM через единый LLM-оркестратор

## 2.2 Не входит в MVP

- Drag-and-drop визуальный редактор интерфейса (GrapesJS)
- Кодовый холст с визуализацией (CodeMirror + Konva) как отдельный экран
- Публичный деплой проектов по внешним ссылкам
- Автоматическая платёжная интеграция (ЮKassa — после MVP)
- Поддержка языков кроме Python в sandbox
- **HTTP-запросы к внешним API из sandbox (`requests`, `httpx`, `beautifulsoup4`) — после MVP**
- **Подключение реального Telegram-бота из sandbox (Режим A шаблона «Telegram-бот») — после MVP**

# 3. Роли и права

## 3.1 Роли

- **Ученик** — создаёт проекты, проходит курсы, общается с ИИ
- **Учитель** — создаёт и назначает курсы, ведёт классы, смотрит и запускает проекты учеников
- **Родитель** — наблюдает прогресс и запускает проекты ребёнка (без доступа к коду)
- **Администратор** — управляет пользователями, курсами, подписками, лимитами LLM

## 3.2 Матрица доступа

| Действие | Ученик | Учитель | Родитель | Админ |
|---|---|---|---|---|
| Регистрация | ✅ | ✅ | ✅ | ❌ |
| Прохождение курсов и тестов | ✅ | Просмотр (preview) | ❌ | ❌ |
| Создание проектов | ✅ | ❌ | ❌ | ❌ |
| Просмотр кода проекта | Свои | Своего класса | ❌ | ✅ |
| Запуск проекта | Свои | Своего класса | Своего ребёнка | ✅ |
| Просмотр аналитики класса | ❌ | Своего класса | ❌ | ✅ |
| Управление курсами | ❌ | Авторские + форки | ❌ | ✅ |
| Управление подписками и токенами | ❌ | ❌ | ❌ | ✅ |

## Модуль M1: Auth & RBAC

### Зона ответственности
Регистрация, вход, управление сессиями, ролевая модель доступа.

### Требования
- Регистрация по email + пароль; обязательные поля формы: имя (`firstName`), фамилия (`lastName`), email, пароль, роль
- Роль выбирается при регистрации; для родителя — привязка через приглашение от ученика
- Подтверждение email (ссылка, TTL 24 часа)
- Сброс пароля через email (ссылка, TTL 1 час)
- JWT: access-токен (15 мин) + refresh-токен (30 дней)
- Автовыход при истечении refresh-токена
- RBAC реализован на уровне бэкенда и фронтенда

## Модуль M2: LMS — Система обучения

### Зона ответственности
Курсы, уроки, тесты, прогресс учеников, онбординг.

### 5.2.0 Онбординг

После первой авторизации ученик видит приветственный экран с тремя шагами-подсказками:

| Шаг | Заголовок | Действие |
|---|---|---|
| 1 | Присоединись к классу | Ввести invite-код класса (выдаётся учителем) |
| 2 | Пройди первый урок | Открыть назначенный курс и начать Урок 1 |
| 3 | Создай свой первый проект | Перейти в AI IDE и выбрать шаблон |

- Онбординг-виджет отображается в кабинете ученика до тех пор, пока все три шага не выполнены
- Каждый выполненный шаг отмечается галочкой
- Ученик может скрыть онбординг вручную в любой момент
- Учитель видит в аналитике, на каком шаге онбординга находятся ученики

Для учителя при первом входе — отдельный онбординг из двух шагов: «Создай класс» → «Назначь курс».

### 5.2.1 Типы курсов

**Системные курсы:**
- Создаются администратором платформы
- Доступны любому учителю для форка и назначения своему классу
- Редактируются только через форк (оригинал защищён)
- Форкнутый курс помечен «На основе: [название оригинала]»

**Авторские курсы:**
- Создаются учителем
- Видны только классам этого учителя
- Полное редактирование без ограничений

**Структура:** Курс → Модуль → Урок

### 5.2.2 Редактор курсов для учителя

- Создание курса: название, описание, обложка
- Добавление модулей и уроков, изменение порядка (drag-sort)
- Редактор урока:
  - Загрузка видео (Selectel Object Storage)
  - Текстовый контент через **WYSIWYG-редактор TipTap**: заголовки H2/H3, жирный, курсив, списки, вставка кода с подсветкой, таблицы, изображения, ссылки
  - Привязка теста к уроку
- Назначение курса одному или нескольким классам
- Предпросмотр курса (кнопка «Просмотреть как ученик»): учитель видит уроки, видео, тексты и тесты в том же интерфейсе, что и ученик; прогресс и результаты тестов при этом не фиксируются

### 5.2.3 Системные курсы платформы

В MVP платформа содержит **один системный курс**, созданный администратором. В дальнейшем библиотека системных курсов будет расширяться.

При создании нового класса учителю автоматически предлагается форкнуть доступные системные курсы — через баннер или модальное окно «Рекомендуемые курсы для нового класса».

#### Курс «Нейросети для IT-проектов: начало»

**Для кого:** школьники без практического опыта работы с нейросетями.
**Цель:** познакомить с возможностями и ограничениями нейросетевых технологий, научить формулировать задачи для ИИ и безопасно использовать результаты в прикладных проектах.
**Структура:** 5 уроков, общая длительность материалов — **225 минут**.

| № | Урок | Краткое описание | Навыки и компетенции | Формат |
|---|---|---|---|---|
| 1 | Что такое нейросети и где они помогают в проектах | Принципы работы нейросетей, реальные кейсы | Выделение задач, границы применимости | Видео + текст + тест |
| 2 | Безопасность, этика и проверка фактов | Конфиденциальность, авторское право, фейки | Цифровая гигиена и правовая грамотность | Видео + текст + тест |
| 3 | Искусство написания запросов | Разбивка задач, шаблоны промтов | Создание эффективных промтов | Видео + текст + тест |
| 4 | Текст и идеи: от темы к структуре и контенту | Генерация идей, описаний, сценариев с ИИ | Генерация и верификация текстов | Видео + текст + тест |
| 5 | Создание ТЗ продукта | Итоговое задание — ТЗ на свой IT-проект | Формирование требований к проекту | Видео + текст + тест |

**Итоговый результат:** учащийся самостоятельно составляет ТЗ на собственный IT-проект с описанием идеи, целей и ожидаемых результатов.

**Ожидаемые результаты освоения курса:**
- определять, когда использование нейросетей целесообразно;
- корректно формулировать промты для разных нейросетевых сервисов;
- проверять достоверность и корректность сгенерированного контента;
- соблюдать принципы безопасности при работе с персональными данными;
- самостоятельно разрабатывать ТЗ для первого проекта на базе нейросети.

### 5.2.4 Настройки класса

| Настройка | По умолчанию | Описание |
|---|---|---|
| Рекомендовать системные курсы при создании класса | ВКЛ | Баннер с предложением форкнуть системные курсы |
| Доступные курсы | Все назначенные | Учитель управляет видимостью курсов для класса |

### 5.2.5 Тесты и прогресс

- Типы вопросов: одиночный выбор (radio), множественный выбор (checkbox)
- Проходной балл: **70% по умолчанию**; учитель может изменить при создании теста (диапазон 50–100%)
- Неограниченные попытки
- После завершения: правильные ответы с пояснениями
- Фиксируется: балл, количество попыток, дата последней попытки
- Прогресс: % завершения на уровне курса/модуля/урока
- Следующий урок открывается только после завершения текущего

## Модуль M3: AI IDE — ИИ-среда создания проектов

### Зона ответственности
Интерактивная среда диалога с ИИ, генерация Python-кода, версионирование, управление шаблонами проектов и промтов.

### 5.3.1 Интерфейс

- **Вверху:** поле ввода промта + селектор модели
- **Слева/по центру:** история диалога с ИИ (чат-интерфейс)
- **Справа:** кодовый редактор (полностью редактируемый) — ИИ выводит сюда результат, ученик может вносить правки вручную в любой момент
- Ответ ИИ выводится **посимвольно** в реальном времени (SSE/streaming)
- Блок кода начинает подсвечивать синтаксис после завершения блока (не посимвольно, чтобы избежать мерцания)
- Кнопка **«Стоп»** — прерывает генерацию с сохранением полученного фрагмента

**Кнопки управления:**
- «Сгенерировать» — отправить промт ИИ
- «Применить код» — сохранить текущее содержимое редактора как новую версию (активна после завершения генерации ИИ **или** после ручного редактирования кода)
- «Запустить» — выполнить код в Python-sandbox
- «Откатить» — выбрать одну из последних 10 версий кода

### 5.3.2 Шаблоны проектов (минимум 5 в MVP)

| # | Шаблон | Описание |
|---|---|---|
| 1 | Чат-ассистент | Диалог с LLM по заданной теме |
| 2 | Генератор текста | Генерация текста по шаблону с переменными |
| 3 | Классификатор/анализатор | Определение тональности, категории |
| 4 | Q&A по тексту | Ответы на вопросы по загруженному тексту |
| 5 | Telegram-бот (симуляция) | Симуляция диалога с ботом внутри платформы |

Каждый шаблон содержит: системный промт, стартовый код-заготовку, описание, теги.

**Telegram-бот — Режим B (Симуляция):**
- Встроенный текстовый интерфейс «вопрос → ответ бота» без выхода в Telegram

### 5.3.3 Библиотека промт-шаблонов

Отдельная от шаблонов проектов библиотека готовых промтов для быстрого старта в AI IDE.

**Назначение:** ученик может выбрать готовый промт из библиотеки, вставить его в поле ввода одним кликом и отредактировать под свою задачу.

**Категории промтов (MVP — минимум 15 шаблонов):**

| Категория | Примеры промтов |
|---|---|
| Генерация кода | «Напиши функцию на Python, которая…», «Создай класс для…» |
| Отладка | «Найди ошибку в этом коде и объясни её», «Почему код выдаёт ошибку…» |
| Объяснение | «Объясни этот код простыми словами», «Что делает эта функция?» |
| Улучшение | «Сделай код чище и добавь комментарии», «Оптимизируй функцию…» |
| Идеи проектов | «Предложи 5 идей для чат-бота на тему…», «Какие функции можно добавить в…» |

**Структура промт-шаблона:**
- Название (до 60 символов)
- Категория
- Текст промта с плейсхолдерами `{{тема}}`, `{{язык}}`, `{{описание}}`
- Теги

**Управление:** промт-шаблоны создаются администратором платформы. Ученики не могут редактировать системные шаблоны, но могут сохранять свои избранные.

**Избранные промты:**
- Рядом с каждым промт-шаблоном отображается кнопка ⭐
- Нажатие на ⭐ добавляет промт в избранное текущего ученика; повторное нажатие — удаляет
- Избранные промты отображаются в отдельной вкладке «Мои избранные» внутри библиотеки промтов (вкладка видна только если в избранном есть хотя бы один промт)
- Вставка промта в поле ввода AI IDE работает одинаково как из общего списка, так и из вкладки «Мои избранные»
- Максимум **50 промтов** в избранном на одного ученика; при превышении показывается предупреждение «Достигнут лимит избранного — удалите ненужные, чтобы добавить новые»
- Если системный промт-шаблон удалён администратором, он автоматически исчезает из избранного всех учеников

## Модуль M4: Python-sandbox

### Зона ответственности
Изолированное выполнение Python-кода, управление пакетами, контроль суточных лимитов, проксирование LLM-вызовов из кода ученика.

### 5.4.1 Среда выполнения

- Изолированный контейнер под каждый запуск
- Python **3.11**
- Без доступа к файловой системе хоста
- Без произвольных сетевых запросов (только через LLM-прокси платформы — `http://litellm:4000`)
- Все исходящие TCP-соединения, кроме адреса LLM-прокси, блокируются на уровне Docker-сети

### 5.4.2 Белый список pip-пакетов

Пакеты указываются в первой строке кода через специальный комментарий:

```python
# pip: openai pandas numpy
```

**Разрешённые пакеты (MVP):**

| Категория      | Пакеты                          |
| -------------- | ------------------------------- |
| Данные         | pandas, numpy                   |
| Утилиты        | python-dotenv, pydantic, loguru |
| Форматирование | rich, tabulate                  |

**Запрещены в MVP:** `requests`, `httpx`, `beautifulsoup4`, `python-telegram-bot`, `langchain-core` (транзитивно устанавливает запрещённый `httpx`) и любые пакеты с прямым доступом к сети, ОС, шифрованием, майнингом.


> **После MVP:** будет добавлена категория «HTTP и парсинг» (`requests`, `httpx`, `beautifulsoup4`), категория «Telegram-боты» (`python-telegram-bot`) и категория «LangChain» (`langchain-core` и производные) — после реализации проверки транзитивных зависимостей при установке пакетов.


### 5.4.3 Лимиты по времени работы (в день)

- Лёгкий проект (консольные, Q&A): до **2 часов** суммарно
- Сложный проект (LLM-чейны, симуляция ботов): до **30 минут** суммарно

Дополнительные ограничения:
- Тайм-аут одного запроса: 30 секунд
- RAM на контейнер: 256 МБ

### 5.4.4 Доступ к проектам

- Сам ученик — полный доступ (код + запуск)
- Учитель класса — просмотр кода + запуск
- Родительский кабинет — только запуск (без кода)

Публичных URL не существует в MVP.

### 5.4.5 Результат запуска

- Консольный вывод — в отдельной панели рядом с редактором
- HTML-вывод — рендер во встроенном iframe

## Модуль M5: LLM-оркестратор

### Зона ответственности
Единый слой маршрутизации всех LLM-запросов платформы, управление провайдерами, кэширование, контроль бюджета.

### 5.5.1 Принципиальные решения

- Используются **только российские LLM** и провайдеры с серверами в РФ
- Зарубежные провайдеры не используются (невозможность легальной оплаты из РФ)
- Все LLM-вызовы проходят через **единый LLM-оркестратор и прокси**
- Ученик **никогда не видит** API-ключи и технические детали
- Токены/кредиты тратятся на: генерацию кода ИИ + LLM-вызовы из sandbox

### 5.5.2 Российские модели

| Отображение в UI | Провайдер | API-модель | Назначение |
|---|---|---|---|
| ⚡ Быстрая | Сбер | `GigaChat-2-Lite` | Быстрые итерации, простые задачи |
| 🧠 Умная | Сбер | `GigaChat-2-Pro` | Сложный и чистый Python-код |
| 🔵 YandexGPT | Яндекс | `yandexgpt-lite` | Объяснения, тексты на русском |
| 🐋 DeepSeek | Timeweb Cloud (РФ) | `deepseek-v3` | Лучшее качество кода |

**Дефолтная модель для генерации кода: DeepSeek V3**
**Дефолтная модель для runtime в sandbox: GigaChat-2-Lite**

### 5.5.3 Актуальные тарифы (март 2026)

| Модель | ₽ / 1К токенов | ₽ / сессию (~2 500 токенов) | Качество кода |
|---|---|---|---|
| GigaChat-2-Lite | 0.022 ₽ | **0.06 ₽** | ⭐⭐⭐ |
| DeepSeek V3 | 0.072 ₽ | **0.18 ₽** | ⭐⭐⭐⭐⭐ |
| GigaChat-2-Pro | 0.167 ₽ | **0.42 ₽** | ⭐⭐⭐⭐ |
| YandexGPT Lite | 0.200 ₽ | **0.50 ₽** | ⭐⭐⭐ |

> **После MVP:** YandexGPT Pro (1.200 ₽ / 1К токенов, ⭐⭐⭐⭐) — планируется к подключению при появлении необходимости в более высоком качестве русскоязычных текстов.

### 5.5.4 Архитектура LLM-слоя

```
[Браузер ученика]
   Промт + модель → /api/llm/generate (SSE)
         ↓
[LLM Orchestrator Service — Node.js]
   1. Проверка JWT и баланса токенов
   2. Формирование system prompt + текущий код
   3. Подсчёт предполагаемых токенов
   4. Запрос в LiteLLM Proxy (stream: true)
   5. SSE-поток → клиент
   6. По завершении: списание токенов, сохранение версии
         ↓
[LiteLLM Proxy — Docker-контейнер]
   OpenAI-совместимый API
   Роутинг по имени модели:
     fast      → GigaChat-2-Lite (Сбер)
     smart     → GigaChat-2-Pro (Сбер)
     yandexgpt → YandexGPT Lite (Яндекс)
     deepseek  → DeepSeek V3 (Timeweb)
   Управление OAuth2 токенами GigaChat (авто-refresh)
   Управление IAM-токенами YandexGPT (авто-refresh)
   Fallback при сбоях (каскадный):
     deepseek → smart → fast → yandexgpt
   Если все модели недоступны → «Сервис временно недоступен»
         ↓
[Провайдеры: GigaChat / YandexGPT / DeepSeek@Timeweb]
```

### 5.5.5 Алгоритм подключения провайдеров

**GigaChat (Сбер):**
1. Регистрация на `developers.sber.ru` через SberID
2. Создание проекта → получение `Client ID` + `Client Secret`
3. Формирование `Authorization Key = Base64(clientId:clientSecret)`
4. Запись в `.env`: `GIGACHAT_CREDENTIALS=<base64>`
5. OAuth2-токены живут 30 минут → LiteLLM делает авто-refresh
6. Добавить сертификат Минцифры РФ в Docker-контейнер (SSL)

**YandexGPT (Яндекс):**
1. Регистрация на `console.yandex.cloud`
2. Создать Folder → создать Service Account с ролью `ai.languageModels.user`
3. Создать API-ключ для Service Account
4. Запись в `.env`: `YANDEX_API_KEY=...`, `YANDEX_FOLDER_ID=b1g...`
5. IAM-токены живут 12 часов → LiteLLM делает авто-refresh

**DeepSeek через Timeweb Cloud:**
1. Регистрация на `timeweb.cloud`
2. Подключить услугу «AI API» → получить API-ключ
3. Endpoint: `https://llm.timeweb.cloud/v1` (OpenAI-совместимый)

## Модуль M6: Личные кабинеты

### Кабинет ученика
- Прогресс по курсам
- Онбординг-виджет (скрывается после выполнения всех шагов)
- Портфолио проектов: карточки с названием, шаблоном, статусом, датой
- Баланс кредитов/токенов, история списаний
- Бейджи-достижения («Первый запуск», «5 проектов», «Завершил курс» и др.)
- Список привязанных родителей с возможностью отозвать доступ
- Привязка через invite-ссылку (максимум 3 взрослых); ссылка действительна **7 дней**, одноразовая
- Кнопка «Пополнить токены»: создаёт заявку `pending` → инструкция по оплате → после подтверждения администратором токены зачисляются

### Кабинет учителя
- Список классов и настройки каждого
- При создании нового класса — баннер с предложением форкнуть системные курсы
- Онбординг-виджет (скрывается после создания класса и назначения курса)
- Таблица учеников: прогресс, количество проектов, активность, результаты тестов, шаг онбординга
- Просмотр проектов учеников с кодом + запуск
- Создание и редактирование авторских курсов, форк системных
- Библиотека сценариев занятий (готовые методички)
- Аналитика класса

### Родительский кабинет

**Концепция:** один кабинет на ученика, несколько взрослых — каждый со своим аккаунтом.

```
Ученик
  └── ParentCabinet (один)
        ├── Мама — User #1 (свой email, пароль)
        └── Папа — User #2 (свой email, пароль)
```

- Все родители видят одинаковый дашборд
- Действия каждого логируются с указанием «кто» (видно ученику)
- Настройки уведомлений — индивидуальные для каждого родителя
- Функциональность: прогресс ребёнка, список проектов + кнопка «Запустить» (без кода), timeline активности за 30 дней, настройки частоты email-отчётов

### Кабинет администратора
- Управление пользователями и ролями
- Управление системными курсами и промт-шаблонами
- Управление подписками: ручная активация тарифов, ручное пополнение токенов, лог операций
- Мониторинг расхода LLM по моделям и пользователям
- Аварийный стоп при аномальном расходе токенов

### Тарифы

| Тариф | Кредиты/мес | Цена | Для кого |

### Механика токенов (MVP)

- Все операции по смене тарифа и пополнению — **вручную администратором**
- Кредит = внутренняя единица платформы (не 1:1 с API-токеном)
- Кредиты тратятся на: LLM-запросы при генерации кода + LLM-вызовы внутри sandbox
- Отображение баланса в шапке ИИ-чата: «потрачено / доступно»

### После MVP — ЮKassa

- Автоматическая оплата через виджет ЮKassa
- Токены зачисляются мгновенно после оплаты через webhook
- `TokenPurchaseRequest.status` → `completed` без участия администратора

# 6. Нефункциональные требования

## 6.1 Производительность

- Время отклика API (не-LLM): ≤ 300 мс (p95)
- Первый токен SSE-ответа от LLM: ≤ 3 секунд
- Запуск Python-sandbox (холодный старт контейнера): ≤ 5 секунд
- Одновременная работа: 100 активных пользователей на MVP-этапе

## 6.2 Надёжность

- Uptime: 99% в рабочие часы (пн–пт, **08:00–22:00 МСК**)
- Резервное копирование БД: ежедневно, хранение 30 дней
- Graceful degradation при недоступности LLM-провайдера: автоматический fallback

## 6.3 Безопасность

- Все данные передаются по HTTPS (TLS 1.2+)
- Пароли хранятся в виде bcrypt-хэша (cost factor 12)
- JWT хранятся в httpOnly cookies (защита от XSS)
- CSRF-защита на всех мутирующих эндпоинтах
- API-ключи провайдеров хранятся только в `.env` / Secret Manager, никогда не передаются клиенту
- Sandbox полностью изолирован: нет доступа к хост-системе и внешней сети

## 6.4 Поддержка браузеров и устройств

- **Поддерживаемые браузеры:** Chrome, Firefox, Safari, Edge (последние 2 мажорные версии)
- Минимальное разрешение экрана: **1280×720** — для полноценной работы трёхпанельного интерфейса AI IDE
- На экранах **1024–1279px** AI IDE переходит в адаптивный режим: вертикальное расположение панелей (история чата сверху, кодовый редактор снизу, консоль вывода — выдвижная панель снизу экрана)
- Остальные разделы платформы (курсы, кабинеты, аналитика) корректно отображаются от **1024px**
- Мобильная адаптация — после MVP

## 6.5 Масштабируемость

- Горизонтальное масштабирование бэкенда через Docker Compose → Kubernetes после MVP
- Статические файлы и видео — через CDN (Selectel Object Storage)
- Redis для кэширования LLM-ответов и хранения сессий

# 9. Пользовательские сценарии

### Сценарий 1: Регистрация ученика и начало обучения

**Участники:** ученик, учитель.
**Предусловие:** учитель создал класс и назначил курс.

1. Ученик открывает платформу → регистрируется (email, пароль, роль «Ученик»).
2. Подтверждает email по ссылке (TTL 24 часа).
3. В кабинете видит онбординг-виджет: шаг 1 «Присоединись к классу».
4. Вводит invite-код класса → присоединяется → шаг 1 отмечен галочкой.
5. Открывает курс «Нейросети для IT-проектов: начало» → проходит Урок 1.
6. Тест: 60% с первой попытки → видит пояснения → повторяет → 80% → шаг 2 отмечен.
7. Создаёт первый проект → шаг 3 отмечен → онбординг-виджет скрывается.
8. Учитель видит в таблице класса: все три шага выполнены.

***

### Сценарий 2: Создание первого проекта учеником

**Участники:** ученик.
**Предусловие:** завершён курс, баланс кредитов > 0.

1. Ученик → «Новый проект» → шаблон «Чат-ассистент».
2. Промт: «Создай бота для изучения английских слов».
3. Выбирает модель «🐋 DeepSeek» → «Сгенерировать».
4. Ответ ИИ приходит посимвольно; код подсвечивается после завершения блока.
5. Вносит правку вручную → «Применить код» → версия №1.
6. «Запустить» → консоль: `Привет! Как переводится слово "apple"?`
7. Новый промт → «Применить код» → версия №2.
8. Сломал логику → «Откатить» → версия №1 → всё работает.

***

### Сценарий 3: Ученик работает с шаблоном «Telegram-бот (симуляция)»

**Участники:** ученик.

1. Новый проект → шаблон «Telegram-бот (симуляция)».
2. Шаблон открывается с пометкой «⚠️ Режим симуляции».
3. Промт: «Бот-помощник по математике для 8 класса».
4. ИИ генерирует код → ученик запускает.
5. В консольной панели: встроенный интерфейс «вопрос → ответ бота».
6. Ученик демонстрирует результат учителю прямо на платформе.

***

### Сценарий 4: Учитель создаёт класс и ведёт работу

**Участники:** учитель, ученики.

1. Учитель регистрируется → создаёт класс «10Б — Информатика».
2. Онбординг учителя: шаг 1 «Создай класс» — выполнен автоматически.
3. Баннер: «Рекомендуемые курсы» → форкает системный курс → шаг 2 выполнен.
4. Редактирует Урок 3 форка через TipTap: добавляет свой пример промта.
5. Назначает курс классу → раздаёт invite-код ученикам.
6. Через два дня открывает аналитику: 14/20 прошли Урок 1, средний балл 78%.
7. Находит проект ученика → открывает код → запускает → убеждается в работоспособности.

### Сценарий 5: Родители подключаются и наблюдают прогресс

**Участники:** ученик, мама, папа.

1. Ученик → «Пригласить родителя» → создаёт invite-ссылку (TTL 7 дней, одноразовая).
2. Мама переходит по ссылке → регистрируется с ролью «Родитель» → привязывается.
3. Ученик получает in-app уведомление → генерирует вторую ссылку для папы.
4. Папа входит в существующий аккаунт → привязывается.
5. Оба видят: прогресс 60%, 2 проекта, timeline активности за 30 дней.
6. Папа нажимает «Запустить» у проекта → видит результат в консоли (без кода).
7. В кабинете ученика: запись «Запустил проект "Чат-ассистент": папа — сегодня, 17:05».
8. Мама настраивает еженедельный email-отчёт; папа — ежедневный.

### Сценарий 6: Пополнение токенов

**Участники:** ученик, администратор.

1. Ученик видит in-app: «⚠️ Остаток кредитов менее 20%».

3. Создаётся заявка `pending` → ученику показывается инструкция по оплате.
4. Ученик делает перевод → отправляет чек администратору.
5. Администратор подтверждает → 50 000 кредитов зачислены.
6. Ученик получает уведомление: «✅ Зачислено 50 000 кредитов».

### Сценарий 7: Аварийный стоп при превышении LLM-бюджета

**Участники:** администратор.

1. LiteLLM достигает лимита 5 000 ₽/мес.
2. Все LLM-запросы автоматически блокируются.

4. Администратор получает срочное in-app уведомление + email.

6. Сбрасывает счётчик → платформа возобновляет работу.
